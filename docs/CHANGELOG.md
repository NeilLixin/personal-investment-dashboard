# Changelog

## v0.7.0 — 2026-07-02

- 新增基金代码候选库、匹配日志和“持仓工作台 → 基金代码补全助手”。
- 候选库按日缓存，AKShare 多数据源逐个隔离失败；支持页面与命令行刷新。
- 名称匹配保留 A/C、QDII、ETF、联接和 LOF 语义；截断、歧义、多候选不会自动写库。
- 代码只有在确认表勾选后才补入空白持仓，已有不同代码受冲突保护。
- 市场刷新缺码提示直接指向补全助手；同步空代码不再覆盖本地已有代码。
- 新增代码规范化、保守匹配、缓存、部分源失败、确认写入和同步冲突测试。

## v0.6.1 — 2026-07-02

- 修复投资日报缺少 `market_snapshot` 时的 KeyError，并提供永久默认结构。
- 统一风险规则的可选 `market_snapshots` 参数，保持旧调用兼容。
- 修复市场品种识别：有效 6 位代码不再因资产类型为空而跳过。
- 拆分真实开放式基金与 ETF AKShare 包装，区分依赖缺失、网络失败和 skipped。
- 刷新结果增加稳定字段、逐条原因、日志与页面高级诊断。
- 新增市场联网测试和 holdings 诊断脚本。

## v0.6.0 — 2026-07-02

- 新增市场快照、刷新日志和收益截图批次表及兼容迁移。
- 新增可选 AKShare 市场刷新；数据库控制一小时刷新间隔，不运行后台任务。
- 新增第三方 App 今日收益截图解析、匹配、中文确认表和人工确认导入。
- 总览、持仓工作台、投资日报和风险雷达接入最新快照。
- 新增市场快照设置；API、OCR 或网络不可用时保持主系统可用。
- 快照不修改持仓成本、交易、计划或复盘，场外基金估算不标记为官方实时净值。

## 2026-07-02 — 文档交接收口

- 重写 AI 交接文档，去除历史流水账、具体持仓、金额和截图隐私。
- 精简项目状态与新窗口上下文，按当前 v0.5.0 代码描述产品方向。
- 明确继续使用 Streamlit，现阶段不进行 React 重写。
- 统一隐私、OCR 人工确认、结构化同步和接手测试原则。

## v0.5.0 — 2026-07-01

- 持仓管理升级为持仓工作台，买卖计划和操作日志合并到持仓详情。

- UI 统一中文、格式化金额收益率并隐藏技术字段。
