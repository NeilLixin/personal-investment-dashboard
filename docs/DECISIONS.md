# Decisions

## 不做自动交易

系统目标是提高记录质量和仓位纪律。自动下单会引入账户安全、错误执行、监管与不可逆损失风险，因此不接券商交易接口。

## 第一版不用行情 API

行情源存在授权、限频、代码映射和错误报价问题。MVP 先验证持仓管理、计划和复盘是否真正有用，数据手动更新或截图导入。

## 使用 Streamlit + SQLite

Streamlit 能快速提供本地中文交互、表格和 Plotly 图表；SQLite 零服务、可备份、适合单人长期使用。

## OCR 是可选依赖

OCR 运行环境较重且可能初始化失败。基础页面必须始终可用，手动粘贴文本是可靠兜底。

## 导入前必须人工确认

OCR 和金融截图解析都可能误识别名称、金额或正负号。任何数据进入持仓前必须让用户修改和选择重复项策略。
